Engineering the dev lifecycle into a "data context layer for AI agents"Personal
  • Precise vs fuzzy memory: the core design thesis—well-defined tasks don't rely on vector similarity recall; analysis.md / plan.md / decisions are persisted as verifiable, re-readable structured files that the next step reads directly
  • Cross-session / cross-AI persistent memory: state lives in the file system, not a single chat window—readable after closing a session or switching AI
  • Engine swapping: skills live in ~/.agents/skills and symlink-sync to Claude / Gemini / Codex, scheduled flexibly by quota, task strength, and cross-verification—no single-vendor lock-in
  • Engineering discipline: task-tiered permissions (read-only Q&A vs write/implement), enforced pre-checks, success judged by persisted files rather than the LLM's self-assessment; includes a review-fix self-correction loop
